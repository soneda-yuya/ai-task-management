## 2. デザイン原則 (Design Principles)

1. **Document-first(ドキュメントファースト)**: チャット UI ではなく、Notion / GitHub PR / AI-DLC の質問ファイルのような **編集可能なドキュメント** を中心に据える。対話もこの中で完結させる
2. **Developer Aesthetic(開発者美学)**: モノスペースフォントのアクセント使用、等幅感のあるカードレイアウト、キーボードショートカット重視(`⌘+K` パレット等)
3. **Less Chrome, More Content(装飾より中身)**: グラデーション・イラスト・過度なアニメーションは避け、余白・タイポグラフィ・アクセント色で情報の階層を表現
4. **Mobile Functional, Desktop Optimal**: デスクトップ 1280px+ が主戦場。モバイルは閲覧 + 軽度な操作(完了切替、コメント)まで動けば可とする
5. **Dark by Default, Light Respects OS**: 起動時は OS のテーマ設定に従う。切替は左サイドバー下部のトグル
6. **Accessibility First**: Radix UI + shadcn/ui が準拠する WAI-ARIA に沿い、キーボード操作・スクリーンリーダー対応を担保

## 3. ビジュアル言語 (Visual Language)

### 3.1 カラー(AI 推奨: モノクロ + アクセント青)

| トークン | Light モード | Dark モード | 用途 |
|---|---|---|---|
| `background` | `#FFFFFF` | `#0A0A0A`(near-black) | ページ背景 |
| `foreground` | `#0A0A0A` | `#FAFAFA` | 本文テキスト |
| `muted` | `#F5F5F5` | `#171717` | サブ領域背景 |
| `muted-foreground` | `#737373` | `#A3A3A3` | 副次テキスト |
| `border` | `#E5E5E5` | `#262626` | 区切り線 |
| `accent / primary` | `#2563EB`(blue-600) | `#3B82F6`(blue-500) | ボタン・リンク・アクセント |
| `accent-foreground` | `#FFFFFF` | `#FFFFFF` | アクセント上テキスト |
| `destructive` | `#DC2626`(red-600) | `#EF4444`(red-500) | 削除・危険操作 |
| `success` | `#16A34A`(green-600) | `#22C55E`(green-500) | 成功・完了 |
| `warning` | `#CA8A04`(yellow-600) | `#EAB308`(yellow-500) | 中リスク・注意 |

**リスクレベルの色付け**(US-3.3):
- 🟢 Low → `success`
- 🟡 Medium → `warning`
- 🔴 High → `destructive`

### 3.2 タイポグラフィ

| 用途 | フォント | サイズ |
|---|---|---|
| UI 全般 | **Inter**(Google Fonts)、フォールバック: system-ui | 14–16px ベース |
| コード・Issue 本文・技術ラベル | **JetBrains Mono** or `ui-monospace` | 13–14px |
| 見出し H1 | Inter Bold | 28px / 1.2 |
| 見出し H2 | Inter Semibold | 22px / 1.3 |
| 見出し H3 | Inter Semibold | 18px / 1.4 |
| 本文 | Inter Regular | 14px / 1.6 |
| キャプション | Inter Regular | 12px / 1.5 |

### 3.3 余白・ラインハイト・角丸・影

- **余白スケール**: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 px(Tailwind デフォルト準拠)
- **ラインハイト**: 本文 1.6、UI テキスト 1.4
- **角丸**: `rounded-md`(6px)を基本、カード `rounded-lg`(8px)、モーダル `rounded-xl`(12px)
- **シャドウ**: ミニマル。デフォルト `shadow-sm`、ホバー `shadow`、モーダル `shadow-lg`
- **ボーダー**: 基本 1px `border-border`、強調時にアクセント色 2px

## 5. レスポンシブ戦略

| ブレークポイント | 幅 | レイアウト |
|---|---|---|
| `sm` | ≥ 375px | モバイル:シングルカラム、サイドバーは下部タブバーに変換、カンバンは横スクロール |
| `md` | ≥ 768px | タブレット:サイドバー折りたたみデフォルト、カンバンは 2 列表示 + 横スクロール |
| `lg` | ≥ 1024px | 小型デスクトップ:サイドバー展開、カンバン 3 列表示 |
| `xl` | ≥ 1280px | **主戦場**:フル表示、カンバン 3 列 + 詳細ペイン併用可能 |
| `2xl` | ≥ 1536px | 広いモニタ:コンテンツ中央寄せ最大幅 1440px |

### モバイル時の主な変換
- サイドバー → ハンバーガーメニュー + 下部 4 タブバー(Tasks / Repos / AI Runs / More)
- カンバン列 → 左右スワイプで切替
- アイデア対話ドキュメント → 全画面表示、キーボード配慮
- AI タイムライン → カード縦積み、展開時は全画面モーダル

### P-3: アイデア対話(AIDLC 形式 = ドキュメント型 Q&A)

**コンセプト**: ChatGPT のようなチャット UI ではなく、**AI が生成した Markdown ライクな質問ドキュメントを、ユーザーがインライン編集する** 形式。AI-DLC ワークフローで本プロジェクト自身を作る体験を、そのままエンドユーザーに提供する。

```
┌──────────┬──────────────────────────────────────────────────┐
│ Sidebar  │  💡 New Idea > "タスク管理アプリを作りたい"      │
│          │  [Save Draft]  [Cancel]  [Submit (⌘+Enter)]     │
│          ├──────────────────────────────────────────────────┤
│          │                                                   │
│          │  # あなたのアイデア                               │
│          │  ┌──────────────────────────────────────────────┐ │
│          │  │ タスク管理アプリを作りたい。AI が仕様を深  │ │
│          │  │ 掘りして自動で実装してくれるようなもの。   │ │
│          │  └──────────────────────────────────────────────┘ │
│          │                                                   │
│          │  ─────────────────────────────────────────        │
│          │                                                   │
│          │  ## AI からの質問(3 件)                          │
│          │                                                   │
│          │  ### Q1. 誰が使うアプリですか?                   │
│          │  ◯ A) 個人用                                      │
│          │  ◉ B) 少人数チーム                                │
│          │  ◯ C) 組織・企業                                  │
│          │  ◯ D) 一般公開                                    │
│          │  ◯ X) Other: [自由記述...........]                │
│          │                                                   │
│          │  ### Q2. どのプラットフォーム?                   │
│          │  ☑ A) Web       ☑ B) モバイル                     │
│          │  ☐ C) デスクトップ ☐ D) CLI                       │
│          │                                                   │
│          │  ### Q3. 最も重視する機能は?(自由記述)         │
│          │  ┌──────────────────────────────────────────────┐ │
│          │  │ (ここに記入...)                              │ │
│          │  └──────────────────────────────────────────────┘ │
│          │                                                   │
│          │  ─────────────────────────────────────────        │
│          │  [🤖 追加で質問して]  [→ タスク提案へ進む]        │
└──────────┴──────────────────────────────────────────────────┘
```

**インタラクション**:
- 各質問は **インライン編集可能**(ラジオ/チェックボックス/テキスト)
- `Submit` で現在の回答を送信 → AI が曖昧さを分析
- 追加質問があれば、**同じドキュメントの下に新セクションとして追記**される(スクロール可能)
- 「→ タスク提案へ進む」で P-4 のドラフトプレビューへ
- 下書き保存で離席 → 復帰可能(Session 状態は C-2 で永続化)

**キーボードショートカット**:
- `⌘+Enter`: 現在の回答を Submit
- `⌘+S`: 下書き保存
- `⌘+→`: タスク提案へ進む
- `Esc`: キャンセル確認ダイアログ

## 9. 特殊 UI パターン仕様

### 9.1 アイデア対話(AIDLC 形式)の状態機械

```
[drafting-idea]
    ↓ (ユーザーがアイデアを入力 + Submit)
[generating-questions]  ←─ AI 生成中(スケルトン表示)
    ↓
[awaiting-answers]      ←─ 質問がドキュメントに挿入され、インライン回答可能
    ↓ (Submit)
[analyzing-answers]
    ↓ 曖昧あり?
    ├─ Yes → 追加質問をドキュメント下部に追記 → [awaiting-answers] に戻る
    └─ No → [generating-proposal]
              ↓
          [draft-ready] → P-4 へ遷移
              ↓ (承認 or 破棄)
          [approved] → タスクをカンバンに作成
          [discarded] → セッション終了(履歴は残す)
```

### 9.2 AI 実行進捗の SSE 更新規約

- チャンネル名: `ai-run:{runId}`
- イベント型: `step.queued / step.running / step.progress / step.succeeded / step.failed / run.completed`
- 再接続時は最新スナップショットを API で取得し、差分のみ SSE で追従

### 9.3 カンバン DnD

- **楽観的 UI 更新**: ドロップ即座に UI 反映
- **Connect RPC**: `TaskService.Move(id, column, position)` を呼び出し
- 失敗時は `toast` + 元位置にロールバック
- サーバー側の衝突は Last-Write-Wins(US-1.10)

## 10. アクセシビリティ方針

- WAI-ARIA: Radix UI 準拠(shadcn/ui 経由で自動適用)
- キーボード操作: 全インタラクションがキーボードで可能
- コントラスト比: 本文 4.5:1、UI 要素 3:1 以上(WCAG AA)
- スクリーンリーダー: `aria-label` を動的要素に付与、SSE 更新は `aria-live="polite"`
- フォーカスリング: Tailwind `focus-visible:ring-2 focus-visible:ring-primary`
- モーションリデュース: `prefers-reduced-motion` に対応、DnD のアニメーションを縮小

## 11. ダークモード実装指針

- shadcn/ui の CSS 変数(`--background`, `--foreground` など)を活用

## 13. 次ステージへの申し送り

### Units Generation
- UI 関連は **U-F(Web Frontend)** にまとめる
- Feature-sliced ディレクトリ構成に本ドキュメントの画面インベントリ(P-1〜P-12)を対応付ける

### Functional Design(per-unit、FE ユニット)
- 各画面のバリデーションルール(文字数、必須項目、日付妥当性)
- キーボードショートカット一覧の整備
- ローディング状態・空状態・エラー状態の UI スペック
- アニメーション仕様(カンバン DnD、タイムラインステップ遷移)

### Code Generation
- shadcn/ui のインストールスクリプトとコンポーネント追加コマンドを初期セットアップに含める
- Tailwind 設定で CSS 変数をトークンとして登録
- `@/components` 直下に `ui/`(shadcn) / `shared/`(共通)/ `features/{name}/components/`(機能固有)の階層を作る
